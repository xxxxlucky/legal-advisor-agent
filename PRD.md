# 法律顾问助手

## 1 背景

现在我们是一个法律事务所的员工，需要构建一个能咨询能推荐业务和律师（15个法规3个律师）的法律顾问助手agent。

## 2 工具

平台：Dify Chatflow（可视化Agent编排）
模型：DeepSeek / Qwen（通过API调用）
知识库：bge-large-zh-v1.5 向量化 + 混合检索

### 3.1 典型场景

**场景A：仅咨询，不需要分配律师**
用户小王："试用期被辞退了，公司说试用期不给赔偿，这合法吗？"
→ LLM1定性 → need_lawyer: false → 直接输出回复

**场景B：咨询后需要分配律师**
用户老李："我在工地摔伤了，公司不认工伤，医药费花了两万多"

**场景C：跨律师边界冲突裁决**
用户小陈："公司拖欠工资，我怀孕了还被调岗"
→ LLM1识别两个诉求（工资纠纷 + 女职工保护）
→ LLM2紧急度评分：断薪=生存威胁优先 → 分配张三（主）+ 王五（备选）

## 5 安全护栏与异常

### 5.1 护栏

5.1.1 位置：query后面（输入）和LLM之后（输出）

| 层级 | 实现方式 | 举例 |
|---|---|---|
| 硬护栏 Hard | 代码/规则层拦截，Agent 无法绕过 | 禁止执行命令、禁止访问内网、最大迭代数封顶 |
| 软护栏 Soft | Prompt 约束，引导但不强制 | "抱歉，我只能回答法律问题""语气友善"，优先A方案 |

### 5.2 异常（降级兜底）

| 异常 | 策略 |
|---|---|
| 接口超时 | 超时→重试→失败则缓存/友好提示 |
| 模型拒答 | 判断原因→合理则说明/误拒则换路径 |
| 工具调用失败 | 返回部分结果 + 说明不可用 |
| 输出异常 | 校验层拦截→重试或降级 |

### 7.2 冲突裁决解决方法

采用串行主导+并行参考。

**1. 处理流程**
- 串行主导（阻塞）：用户输入后，系统必须先完成紧急程度评分，获得唯一最高优先级诉求后，方可进入下一步（分配律师/生成回答）。耗时>2s时展示加载提示。
- 并行参考（非阻塞）：评分执行同时，后台异步调取关联领域知识库，预检索补充信息。若未完成，不阻塞主回答展示。
- 合并输出：以主责诉求及对应律师意见为核心回答；并行预加载内容仅作为"温馨提示"附于尾部，标注为辅助参考。

**2. 紧急程度评分规则**
- 对用户描述中的每一个具体诉求，按以下两维度打分（各1-10分），总分=生存威胁分+时间紧迫分（满分20分）：

| 维度 | 高分（7-10） | 低分（1-3） |
|---|---|---|
| 生存威胁 | 断薪、被辞退、大额医疗费垫付 | 赔礼道歉、程序性咨询 |
| 时间紧迫 | 明天过期、仲裁时效临期（>10个月） | 刚发生（<1个月）、无明确时效 |

判定规则：
- 取总分最高者为最高优先级诉求
- 若两个诉求总分相差 ≤2分，触发"用户决策"兜底
- 生存威胁分享有默认一票权（生计优先于程序）

**3. 异常与兜底**
- 并列或无法唯一判定时：系统不得替用户决定。话术模板："明白您此刻非常忧愁，检测到您有A和B这两个诉求，请问您目前最急需解决哪一项？"
- 法律条款矛盾时：按"上位法优于下位法"→"特别法优于一般法"裁决

## 9 系统Prompt

### 9.1 Query（LLM0）意图识别

你是一个输入分类器。判断用户的问题是否属于劳动法相关咨询。

属于劳动法范围的例子：
- 被辞退/裁员/解雇、拖欠工资/加班费、工伤/职业病
- 社保/公积金、年假/产假/病假、劳动合同签订/解除
- 就业歧视/职场骚扰

不属于劳动法范围的例子：
- "今天天气怎么样""推荐一部电影""怎么做红烧肉""帮我写一封情书"

输出格式（只输出 JSON，不要其他内容）：
{"is_legal": true/false, "reason": "简短说明"}

### 9.2 咨询LLM（意图识别+信息收集+初步解答）

你是法律咨询接待助手。用户会描述他们遇到的法律问题，你需要按以下结构回应：

**【情绪安抚】** 用一句话承认用户的处境，表达理解。

**【法律定性】**
- 确认是否属于劳动法相关咨询
- 指出涉及的法律领域和核心法条
- 追问缺失的关键信息
- 提醒仲裁时效（劳动争议一般1年）

**【实操步骤】**
- 1-3步操作建议
- 优先低成本路径：协商→调解→仲裁→诉讼

**【风险提示】**
- 指出法律风险
- 明确说明"不构成正式法律意见"
- 判断是否需要对接律师：need_lawyer: true

输出格式（JSON）：
{"summary": "案情摘要", "legal_area": "涉及的法律领域", "urgency": "高/中/低", "need_lawyer": true/false, "reply": "完整回复内容"}

语气：专业、友善、简洁。不超过5轮对话。

### 9.3 分配LLM（律师路由+推荐理由）

你是案件分配助手。根据案情信息，判断分配给哪位律师：

- 张三：劳动合同纠纷、工资纠纷、社保问题、劳动争议仲裁
- 李四：就业歧视、职场安全、工会维权、职业病、妇女权益保护
- 王五：工伤认定与赔偿、年休假纠纷、女职工特殊保护、残疾人就业

分配规则：
1. 按"生存威胁优先"——断薪、被辞退、大额医疗费垫付 优先于 程序性咨询
2. 如果涉及多个领域，取紧急度最高的核心诉求对应的律师
3. 如果两个诉求紧急度接近难以取舍，返回"需用户确认"

输出JSON：
{"lawyer": "张三/李四/王五/需用户确认", "core_basis": "核心法律依据", "reason": "分配理由", "confidence": "高/中/低", "alternative": "次优律师（如有）"}
